Java Tools Required for Understaning Electron Devices

The text uses a protocol for delivering executable Java codes called Java Web Start to launch the live figures.  Oracle (which owns the original Sun Microsystems Java rights) removed this facility after Java version 1.8 (in 2017).    Fortunately, an open-source version of Web Start has recently (as of 2021)  become available.  Please download it f rom: https://openwebstart.com/download/ and install it on your system.  Open Web Start (OWS) will then be associated with the *.jnlp file-type, and will atuomatically launch the program codes specified in the jnlp file.  OWS invokes the Java Runtime Environment (JRE) that is installed on your computer.

Adobe has also disabled the mechanism that permits external files to be launched from within Acrobat Reader.  The best solution that I have found is to use the Foxit PDF reader instead, to view the textbook (UndElDev.pdf) and the lecture slides (UEDfoils.pdf).  The Foxit reader can be downloaded from: https://www.foxit.com/downloads/ .  Install this software, open the reader and go to File -> Preferences -> Trust Manager and uncheck Enable Safe Reading Mode.  

Some figures require the Java3D extensions, which should be automatically downloaded to your computer through the OWS system.  There may be an issue with the 32 versus 64 JRE software.  If you have problems with a 64-bit installation, try the 32-bit version of Java.

Users of Apple computers may encounter security restrictions placed upon executable codes that have not been obtained through the Apple website.  To date this seems to be a problem with codes that are directly run from a Java executable (JAR) file, but I would not be surprised to see problems with OWS in the future.  Instructions on how to disable these protections can be found at: https://support.apple.com/en-us/HT202491 .
